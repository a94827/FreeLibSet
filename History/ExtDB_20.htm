21.04.2023-26.05.2023

1. Изменения в конструкторе DBxColumns (v.1 п.2)

2. Внутренние изменения в методах DBxColumns, DBxColumnList()

3. Внутренний класс DBxTableColumnList.TableCollection не был помечен [Seializable]

4. В методе DBxExpression.GetValue() убран аргумент nullAsDefaultValue (v.45 п.1)

5. Изменения в DBxFunction (v.45 п.1)

- Убраны неподдерживаемые неагрегатные функции DBxFunctionKind.Min и Max

- GetValue() для функции DBxFunctionKind.Substring реализовано через DataTools.Substring()

- GetValue() возвращает null, если хотя бы один из аргументов равен null

- GetConst() возвращает константное выражение, если все аргументы являются константными.

6. Изменения в CompareFilter (v.45 п.10):

- Конструктор проверяет корректность аргументов Kind и NullAsDefaultValue, когда один из аргументов - константа NULL.

- Свойство ComparisionToNull

- TestFilter() обрабатывает сравнение с константами DBxConst(null), аналогично SQL-запросам IS NULL/IS NOT NULL

- TestFilter() использует DataTools.CompareNumbers() (ExtTools, v.345 п.6)

7. Конструкторы ValueFilter() правильно устанавливают свойство NullAsDefaultValue для операций сравнения, отличных от Equal. Выделен protected-метод DBxSqlFormatter.GetDefaultValue(). Метод переопределяется в SQLite, где минимальной датой считается '1000-01-01'

8. Изменения в ValuesFilter (v.1 п.3):

- Конструктор проверяет, что массив не содержит значений null и DBNull и все значения имеют один тип.

- Свойство NullAsDefaultValue (только для чтения)

- TestFilter() использует DataTools.CompareNumbers() (ExtTools, v.345 п.6)

9. Метод SubstringFilter.TestFilter() (v.1 п.3) был реализован неправильно.

10. Изменения в BaseDBxSqlFormatter (v.45 п.3):

- OnFormatExpression() для DBxConst приоритетно использует columnType из DBxFormatInfo, а не из DBxConst.

11. Изменения в DataViewDBxSqlFormatter (v.1 п.5):

- OnFormatColumn() использует свойство DBxSqlBuffer.ColumnStructs, как метод BaseDBxSqlFormatter.OnFormatColumn().

- Реализован метод OnFormatFunction() для функции DBxFunctionType.Abs через IIF().

12. Функция DBxTools.Convert(). Используется методами CompareFilter, ValuesFilter.TestFilter().

13. Определено, что для DBxColumnType.Time основным типом является TimeSpan, а не DateTime. Изменения в методах DBxTools.

14. Изменения в SQLiteDBxSqlFormatter (v.29 п.1):

- Функция DBxFunctionKind.Length имеет имя "LENGTH", а не "LEN"

- Поддержка типа TimeSpan в OnFormatCompareFilter() (только для сравнения "=" и "<>") и OnFormatValuesFilter. Используется преобразование в строку с помощью "CONVERT(ИмяПоля,System.String)", т.к. DataView не поддерживает операции сравнения для типа TimeSpan

- Поддержка типа Guid в OnFormatValuesFilter() (для CompareFilter сравнение работает)

15. DBxFilter.GetAllFilters() (v.48 п.5) принимает аргумент IList<DBxFilter>, а не List<>

16. В методах AndFilter, OrFilter.FromArray(), FromList() исправлен поиск "вырожденных" фильтров (v.36 п.6). Исправлена реализация оператора "|" для DBxFilter для "вырожденных" фильтров.

17. Черновая поддержка временных таблицы в MS SQL Server (v.1 п.11). Метод SqlDBxCon.CreateTempTableInternal(). Переопределен метод SqlDBx.IsValidTableName() для разрешения таблиц, начинающихся с "#". Виртуальный метод DBxConBase.GetTableStruct(), переопределен в SqlDBxCon.

18. Перегрузка DBxTableStruct.Clone() (v.1 п.1) с именем таблицы.

19. Конструктор DBxNameValidator (v.45 п.9) изменен и сделан internal.
