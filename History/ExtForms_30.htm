27.09.2024-18.11.2024

1. Перенесено из ExtDBDocsForms:

- EFPDBxGridFilterSet (был EFPClientFilterSet, ExtDBDocsForms, v.102 п.7)

- EFPDBxGridFilters (объединяет в себя все из DBxClientFilters и GridFilters, ExtDBDocsForms, v.101 п.1 больше нет разбиения на отдельные классы)

- Классы фильтров StringValueGridFilter, StartsWithGridFilter, EnumGridFilter, BoolValueGridFilter, IntValueGridFilter, NullNotNullGridFilter, DummyGridFilter, FixedSqlGridFilter, Int/Single/Double/Decimal/DateRangeRangeGridFilter/InclusionGridFilter/RangeCrossGridFilter (ExtDBDocsForms, v.1 п.6, v.101 п.2)

2. Свойство StringValueGridFilter.UseManualInput (ExtDBDocsForms, v.1 п.6). Свойство ValueList переименовано в AllValue. Событие ValueListNeeded – в AllValuesNeeded.

3. Свойство EFPGridFilters.DataViewRowFilter (v.351 п.23)

4. Класс StringValuesGridFilter

5. Событие EFPDataGridView, EFPTreeViewAdv.AfterRefreshData. Используется в EFPGridFilterGridView (v.89 п.3) вместо события RefreshData, чтобы не появлялась команда «Обновить данные»

6. Изменена реализация EFPFormChecks и свойства EFPBaseProvider.FormChecks (v.289 п.1) для совметимости cо стандартным интерфейсом ICollection. Методы EFPFormProvider.AddFormCheck() заменены на перегрузки EFPBaseProvider.FormCheckCollection.Add().

7. Метод EFPDataGridView.IndexOfUsedColumnName() (v.276 п.8) переименован в IndexOfUsedColumn(). Возвращается первый видимый столбец из EFPDataGridViewSortInfo.ClickableColumnNames, а не столбец, находящийся первым слева.

8. Установка свойства EFPStdConfigurableDataGridView.GridProducer не устанавливает AutoSort=true (v.291 п.5), если EPGridProducer.OrderCount=0.

9. В методах EFPGridProducerColumn.ApplyConfig() (v.276 п.9) предотвращается возникновение исключения при установке свойства Width, если оно меньше текущего значения MinimumWidth

10. ApplicationExceptionHandler (v.137 п.1) перехватывает исключение ArgumentException в конфигурации Wine+Mono, которые выбрасывается при прорисовке всплывающей подсказки для кнопки панели инструментов.

11. При отправке табличного / иерархического просмотра в Excel / Calc (v.348 п.33, 34) показываются флажки «Заголовок таблицы» и «Фильтры» в блоке «Что выводить». Свойства BRDataViewSettingsDataItem.ExpTableHeader, UseExpTableHeader, UseExpTableFilters, ExpTableFilters. Такой же диалог показывается при сохранении просмотра в форматах Excel, Word, ODS и ODT.

12. Для таблицы редактирования фильтров EFPGridFilterEditorGridView (v.231 п.2) настроены параметры печати по умолчанию

13. Для табличных и иерархических просмотров — команда локального меню «Настройка форматов копирования». Свойства EFPDataGridViewCommandItems, EFPTreeViewCommandItemsBase.SelectedCopyFormats (v.79 п.2).

18. Исправлена реализация копирования данных в буфер обмена в формате CSV. Метод WinFormsTools.SetTextMatrix() (v.301 п.23) разделен на два: SetTextMatrixText() и SetTextMatrixCsv()

19. Исправлена реализация получения текстового значения ячейки табличного просмотра (внутренний метод EFPDataGridView.DoGetCellTextValue()). Если EFPDataGridViewCellAttributesEventArgs.FormattingApplied=false, а значение ячейки является IFormattible, то используется DataGridViewCellStyle.Format и FormatProvider.

20. Альтернативная возможность добавление команд меню в «уголочек» панели инструментов (v.207 п.5) без объявления подменю. Флаг EFPCommandItemUsage.ToolBarAux.
