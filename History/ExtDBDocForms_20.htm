311

17.02.2022-24.02.2022

1. Еще раз исправлена установка фокуса ввода в DocTableViewForm (v.134 п.1). Вызовы SetFocus() (исправлено ExtForms, v.311 п.2) выполняются в OnVisibleChanged(), а не OnActivated(). Предыдущая реализация иногда не срабатывала, зато активировала просмотр таблицы или дерева при переключении окон.

2. Свойство EFPDBxTreeView.SelectedIdsWithChildren (v.1 п.4)

3. Изменена реализация чтения свойства EFPDBxTreeView.SelectedIds. Теперь используется свойство SelectedDataRows. В возвращаемом массиве не могут появляться идентификаторы 0.

4. Изменения в моделях деревьев DataTableTreeModelWithIds<Int32> (ExtTools, v.311 п.2).

5. EFPDocTreeView (v.1 п.4) в режиме удаления пытается удалять документы вместе со всеми вложенными документами. Раньше такая попытка удаления была обречена на провал.

6. EFPDocTreeView.IntDocumentViewHandler.ApplyChanges() выполняет позиционирование после редактирования документов. В процессе редактирования могла поменяться структура дерева. Аналогично, после выполняется позиционирование после редактирования поддокументов в EFPSubDocTreeView.OnEditData().

7. Метод EFPDocTreeView, EFPGroupDocTreeView.IntDocumentViewHandler.CopyRowValues() никогда не вызывался (v.100 п.6)

8. Внутренний метод EFPGroupDocTreeView.AddRootNode() сначала добавляет строку в таблицу, а потом меняет остальные ссылочные поля. Иначе возникает исключение в DataTableTreeModel.

9. Изменения в переходниках DocValueControlBase для использования DBxDocValue.ValueArray вместо ComplexValue (ExtDBDocs, v.311 п.11)

10. Метод InitDocEditFormEventArgs.AddRefToParent(), который дополнительно добавляет валидатор к EFPDocComboBox для предотвращения зацикливания дерева документов.

310

03.02.2022 (вечер) - 04.02.2022

1. Методы DBUI.UpdateDBCacheAndRows(), DocumentViewHandlerList.UpdateDBCacheAndRows() (v.106 п.1) и DocProviderUI.AfterApplyChanges() вызывают DocTypeUI.RefreshBufferedData(). Для справочников, с установленным свойством DocTypeUI.DataBuffering=true не выполнялось обновление буфера справочника при добавлении элементов (и других действиях). При следующем открытии справочника новый элемент не отображался в просмотре до выполнения команды "Обновить".

2. EFPGroupDocTreeView обрабатывает OnAttached(), где заново инициализирует текст для корневого узла дерева. Создание модели и корневого узла (v.101 п.4) может выполняться до инициализации списка столбцов, когда нельзя установить текст.

3. EFPDocGridView, EFPDocTreeView.OnCreated() перехватывают ошибки при вызове PerformRefresh(). Если исключение "выпустить" наружу, то метод OnCreated() мог быть вызван еще раз, что приводило к ошибке повторной инициализации меню.

308

07.01.2022-29.01.2022

1. Классы DocValueInt/Single/Double/DecimalEditBox (v.301 п.2) присоединяются к свойству NValue управляющего элемента, чтобы поддерживать пустые значения, когда EFPNumEditBoxBase<T>.CanBeEmpty=true

2. Свойство InitDocEditFormEventArgsBase.MainImageKey (v.1 п.10), возвращающий значок для документа/поддокумента. При добавлении первой страницы, метод AddPage() устанавливает DocEditPage.ImageKey. Для второй и последующих страниц используется значок "Properties".

3. Свойства EFPMultiDocComboBox (v.71 п2), EFPMultiSubDocComboBox.SelectionMode (v.94 п.3, v.133 п.4)

4. EFPDocTableView.Init() (v.85 п.1) инициализирует EFPDBxGridView.MarkRowIds при Mode=DocTableViewMode.SelectMultiWithFlags. Не работал выбор документов с помощью флажков для DocSelectDialog.SelectionMode=MultiCheckBoxes.

5. Изменения в EFPSubDocGridView, EFPSubDocTreeView:

- Установка свойства ValidateBeforeEdit в true не допускается, если просмотр не относится к редактору документа

- Обработчик события DocumentEditor.BeforeWrite (только EFPSubDocGridView) присоединяется только на время нахождения на экране, а не в конструкторе, чтобы предотвратить утечку ресурсов при многократном создании просмотров.

6. Провайдер комбоблока EFPAllSubDocComboBox (черновик, недоделано)

7. Исправлена инициализация фокуса ввода в EFPDocTableView. Раньше всегда активировался табличный просмотр, даже если в предыдущем сеансе была выбрана вкладка "Дерево".

8. Свойства EFPDocGridView, EFPDocTreeView, EFPSubDocGridView и EFPSubDocTreeView.DefaultyConfiogName и UserInitData сделаны устанавливаемыми. Убраны конструкторы с этими аргументами (v.95 п.1)

9. DocTypeUI, SubDocTypeUI.PerformInitGrid() (v.1 п.1) устанавливают для всех столбцов табличного просмотра, кроме столбца флажков, если он есть, признак ReadOnly.
